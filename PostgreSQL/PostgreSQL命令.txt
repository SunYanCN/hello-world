
1. 添加主键
alter table goods add primary key(sid);

2. 添加外键
alter table orders add foreign key(goods_id) references goods(sid)  on update cascade on delete cascade;

on update cascade: 被引用行更新时，引用行自动更新； 
on update restrict: 被引用的行禁止更新；
on delete cascade: 被引用行删除时，引用行也一起删除；
on dellete restrict: 被引用的行禁止删除；

3. 删除外键
alter table orders drop constraint orders_goods_id_fkey;

4. 添加唯一约束
alter table goods add constraint unique_goods_sid unique(sid);

5. 删除默认值
alter table goods  alter column sid drop default;

6. 修改字段的数据类型
alter table goods alter column sid type character varying;

7. 重命名字段
alter table goods rename column sid to ssid;

-- 用户对某表权限的撤销及恢复：
撤销 TRUNCATE 权限	REVOKE TRUNCATE ON table_name1 FROM user3;
恢复 TRUNCATE 权限	GRANT TRUNCATE ON TABLE table_name1 TO user3;

-- 禁止某个用户对某表删除操作
-- 1. 查看当前权限
\z+ public.table_name1

-- 2. 撤销 TRUNCATE 权限
REVOKE TRUNCATE ON TABLE public.table_name1 FROM user3;

-- 3. 撤销 DELETE 权限（如果需要）
REVOKE DELETE ON TABLE public.table_name1 FROM user3;

PostgreSQL 中 不能直接 REVOKE DROP；单将表的所有权转移给其他用户（如管理员），这样，user3 用户将不再拥有该表的所有权，也就无法执行 DROP TABLE table_name1。
-- 4. 转移表所有权（禁止 DROP）
ALTER TABLE public.table_name1 OWNER TO postgres;

-- 5. 重新授予 SELECT 等权限
GRANT SELECT, INSERT, UPDATE,REFERENCES,TRIGGER ON TABLE public.table_name1 TO user3;

你可以使用以下命令查看当前 user3 用户对表的权限：
\z table_name1
输出示例：
                          Access privileges
 Schema |         Table          |  Access privileges
--------+------------------------+-----------------------
 public | table_name1   | user3=arwdDxt/owner
a = INSERT
r = SELECT
w = UPDATE
d = DELETE
D = TRUNCATE
x = REFERENCES
t = TRIGGER
如果 D（TRUNCATE）或 d（DELETE）存在，说明用户有权限。
